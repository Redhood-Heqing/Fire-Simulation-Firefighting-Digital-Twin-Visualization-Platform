# 答辩演示脚本

## 1. 正常运行演示

1. 打开 `http://localhost:5173/dashboard`。
2. 介绍中央三维图书馆数字孪生场景。
3. 展示左右两侧实时数据：通行人数、建筑内人数、设备完好率、能耗、环境、火灾风险。
4. 点击传感器点位，查看空间标签和实时数据。
5. 展示后台数据流滚动。

## 2. 模型编辑演示

1. 进入 `/model-editor`。
2. 点击图书馆主楼。
3. 在属性面板修改名称、颜色、透明度或高度。
4. 在场景中拖动建筑位置。
5. 点击“保存构件”。
6. 切换楼层显示、透明模式和消防视图。

## 3. 传感器点位演示

1. 进入 `/sensors` 或 `/model-editor`。
2. 选择传感器类型，例如 `temperature`。
3. 在三维场景中点击放置新传感器。
4. 选中新点位，修改名称、楼层、所属区域和采样频率。
5. 拖动点位位置后保存。

## 4. 火灾应急演示

1. 进入 `/fire-emergency`。
2. 点击“模拟火灾”。
3. 说明系统已生成起火点、红色火灾标记、风险区域和烟雾体积。
4. 观察温度、烟雾、风险指数快速升高。
5. 展示 AI 建议，包括排烟、断电、防火门和疏散建议。
6. 展示推荐路线 A 与备用路线 B。
7. 点击“开启排烟系统”，说明扩散范围与路线成本会变化。
8. 点击“结束演示”，系统恢复正常。

## 5. 结束说明

强调本系统采用模拟感知数据、三维数字孪生场景和应急算法推演构建工程化原型，可继续接入真实硬件、真实 BIM 模型和正式消防系统接口。
