# 火灾推演与逃生路线算法说明

## 火势蔓延

后端使用简化网格算法进行演示型推演：

- 每层被划分为二维网格。
- 起火点所在网格首先进入高危状态。
- 每秒推进一次 `spreadStep`。
- 距离起火点越近，温度、烟雾、风险值越高。
- 火灾层扩散最快，相邻楼层受烟气影响。
- 开启排烟系统后，扩散半径降低。

网格状态：

- `safe`：安全区域
- `warning`：警戒区域
- `danger`：高危区域
- `blocked`：禁行区域

## 传感器联动

火灾发生后，传感器读数会根据与起火点的距离变化：

- 温度快速升高
- 烟雾浓度快速升高
- AI 风险指数升高
- 周边声光报警器进入 `active`
- 状态切换为 `warning` 或 `danger`

## 受影响区域

系统根据火灾网格生成：

- 当前受影响楼层
- 高危区域
- 警戒区域
- 禁行区域
- 预计影响人数
- 风险等级

接口：

```http
GET /api/fire/affected-zones
```

## 逃生路线

路线规划使用 Dijkstra 成本算法。

节点包括：

- 3F 东侧阅览区
- 中央走廊
- 南侧走廊
- 西侧安全楼梯
- 南侧楼梯
- 西门出口
- 南门出口
- 室外安全集合点

规则：

- 普通边使用基础距离成本。
- `warning` 区域提高通行成本。
- `danger` 和 `blocked` 区域施加高成本或禁止通行。
- 系统输出推荐路线 A 与备用路线 B。

接口：

```http
GET /api/fire/routes
```
